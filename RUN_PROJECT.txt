Database setup
--------------
The backend uses MongoDB through Mongoose.

Default local database URL:
  mongodb://127.0.0.1:27017/hospital_management

The backend reads this value from:
  backend\.env

Expected backend .env:
  MONGODB_URI=mongodb://127.0.0.1:27017/hospital_management
  JWT_SECRET=your_secret_key_here
  JWT_EXPIRY=7d
  PORT=5000
  NODE_ENV=development

Optional CORS setting if needed:
  FRONTEND_URL=http://localhost:5173,http://127.0.0.1:5173

To start local MongoDB on Windows, open a terminal and run:
  mongod

If MongoDB is installed as a Windows service, start it from Services, or run:
  
  

To check if MongoDB is running in PowerShell:
  Get-Process | Where-Object { $_.ProcessName -match 'mongod|mongo' }

To connect with MongoDB Shell:
  mongosh

Then inside mongosh:
  use hospital_management
  show collections

MongoDB creates the hospital_management database automatically after the app saves data.

API connection
--------------
The frontend API base URL is defined in:
  Hospital\src\services\api.js

Default API URL:
  http://localhost:5000/api

Optional frontend .env file:
  VITE_API_URL=http://localhost:5000/api

If the backend port changes, update VITE_API_URL or the default value in api.js.

Common problems
---------------
Problem: frontend cannot call backend.
Check:
  1. Backend is running on port 5000.
  2. Frontend is running on port 5173.
  3. backend\.env FRONTEND_URL includes the frontend URL.
  4. Hospital\src\services\api.js points to http://localhost:5000/api.

Problem: login/signup fails with database errors.
Check:
  1. MongoDB is running.
  2. backend\.env has MONGODB_URI.
  3. Backend terminal says "MongoDB connected successfully".

Problem: port already in use.
Check listening ports in PowerShell:
  Get-NetTCPConnection -State Listen | Where-Object { $_.LocalPort -in 5000,5173 }

Problem: dependencies are missing.
Run npm install inside both backend and Hospital folders.
